Docker学习

发表于 2018-06-04 | 分类于 DevOps

Docker学习之Dock初识（一）

DockerFile中的命令详解

ENV 命令用来指定环境变量

ex:

  1. ENV TZ “Asia/ShangHai” 用来指定镜像的时区
  2. ENV TREM xterm 用来指定Linux的虚拟终端类型

ADD和COPY指令

  1. 需要说明的是在DockerFile中，ADD和COPY指令都用来拷贝文件，区别在于ADD可以自动将文件解压后拷贝，而COPY指令只能将解压后的文件进行拷贝。
  2. 一般ADD命令用来添加一些镜像源或者拷贝一些配置文件，例如在Centos7中：
  • ADD aliyun-mirror.repo /etc/yum.repos.d/CentOS-Base.repo
  • ADD aliyun-epel.repo /etc/yum.repos.d/epel.repo

VOLUME 指令

  1. 通过宿主机映射contina中的文件夹

ONBUILD 指令

  1. ONBUILD RUN 指令用来当构建这个镜像时，指定的命令不会去执行
  2. 通过当前镜像作为基础镜像，构建下一个镜像时，启动生效。

RUN 指令

  1. RUN指令后一般会书写可被执行的SHELL命令。
    ex:
    • yum install lzzrz(安装文件上传下载服务)
    • bash -c ‘touch /app.jar’(拷贝编译后的jar到容器中)

EXPOSE 指令

1.在DockFile中用EXPOSE：端口号 可以选择将指定的端口号暴露给宿主机

ENTRYPOINT 指令

1.ENTRYPOINT 指令用来指定容器（Continer）每次启动的时候执行的命令
ex:

  • ENTRYPOINT [“/usr/bin/supervisord”, “-n”, “-c”, “/etc/supervisord.conf”] 这个命令就是用来在容器启动的时候，启动进程管理器 Supervisord 并指定配置文件

Docker 命令相关

Docker Build

  1. Docker build -t repository_url/namespace/continer_name/centos-lastest .

-t 代表指定镜像的名称
不写 -t 参数 则需指定 /path DockFile 文件所在的位置
最后的. DockFile在当前目录下

Docker run

1.docker run -d -p 2222:22 –name dbserver

  • -d 在后端启动，返回docker的lear
  • -it 以交互模式启动
  • -p 映射端口，不管continar 是否重启映射端口不会发生变化
  • -P（大写）宿主机会随机选择端口映射22，重新启动之后会发生变化

Docker ps

  1. docker ps 查看所有正在运行的容器。
  2. docker ps -a 查看所有已经构建成功的容器。

###遇到的一些问题及解决方案

  1. 在写Docker中的启动脚本时要在SHELL脚本的开始加上 set -e
  2. set -e 的作用是当脚本执行发生错误时，立即停止执行，避免造成更大的影响
  3. 在windows中执行Shell脚本的时候，会因为windows 和 unix 的换行符和其他隐藏字符导致脚本无法继续执行，此时要通过 :set ff 查看当前文档的格式是unix 还是windows，如果是windows 需要更改为unix,保存之后，继续执行就不会发生错误了。

MongoDB学习笔记（一，安装MongoDB&可视化工具Compose）

Windows（MongoDB）下的安装

1.首先从官网下载MongoDB的zip包，解压到指定的目录，在创建两个文件夹分别是MongoDBDATA,MongoDBLOG,这两个文件夹可以随意命名，分别用来存储数据和日志文件，为了见名知义，我用了上面的命名。在MongDBLOG这个文件夹下创建一个mongodb.log文件，这个文件的作用是用来保存MongoDB的日志文件。

2.接下来使用CMD，进入到MongDB的bin目录下，执行mongod –dpath:（D:MongoDB\MongoDBDATA）此处的路径与上面你创建的文件夹目录一致即可。

3.在MongoDB的安装路径下，创建mongod.cfg（采用了YAML的文件格式）, 使用SubLine 打开后编辑内容
systemLog: (指定MongoDB的log文件存储的位置)
destination: file
path: D:\MongoDB\MongoDBLOG\mongod.log
storage: （指定MongoDB的数据存储的位置）
journal:
enabled: true
dbPath: D:\MongoDB\MongoDBDATA
bind_ip: (指定MongoDB的绑定IP，此处指定为0.0.0.0为任意IP都可访问)
0.0.0.0
net:
port:27017(指定MongoDB的端口)
security:
authorization:enable(启用认证！)

4.设置新的环境变量MONGO_HOME，即MongoDB的安装目录，在PATH环境变量下加入%MONGO_HOME%\bin, 这一步的作用是让Mongo命令可以在任意盘符使用。

5.接下来安装MongoDB的服务，使其开机自启动，避免每次启动的麻烦，也可以省略这一步。

6.使用管理员身份打开CMD，切换到MongoDB的bin目录下，使用下面的命令

mongod –config D:\MongoDB\mongod.cfg –serviceName MongoDB –install

7.安装完成之后，可以通过Windows的服务管理器设置为开机自启动，就可以避免每次手动启动MongoDB的服务了。

Windows(MongDB Compose)可视化图形管理工具的安装

1.首先在MongoDB的官方网站上下载MongoDBCompose的安装包
https://www.mongodb.com/download-center?jmp=blog#compass

2.双击这个exe程序，完成安装即可。

单例设计模式

发表于 2016-11-04 | 分类于 设计模式

常见的单例设计模式写法

  • 这种单例模式不仅不是线程安全的,而且通过反射就可以拿到实例对象
#懒汉式
      private static Singleton mySingleton = null;

        /**
         * 私有化构造函数,防止通过构造函数创建对象
         */
        private Singleton(){}

        public  static Singleton getMySingleton(){

            if (mySingleton == null){
                return new Singleton();
            }else {
                return mySingleton;
            }

        }



#饿汉式
 private static Singleton mySingleton = new Singleton();

    /**
     * 私有化构造函数,防止通过构造函数创建对象
     */
    private Singleton(){}

    /**
     * @return 直接返回实例对象
     */
    public  static Singleton getMySingleton(){
        return mySingleton;
    }

  • 但是上述写法不是线程安全的，假如有两个线程同时访问getMySingleton,因为Instance是空，所以同时执行了new Instance，被构建了两次。

#SSH 面试题相关

##Oracle常见面试题
1.Oracle 中的分页 与 Mysql的分页都是如何实现的 ?

Mysql的分页很简单, 通过limit 关键字就可以实现分页;
`select * from tables limit 开始查询的记录,查询的条数`

而Oracle的分页相对来说比较麻烦, 需要通过rownum 伪列, 配置三层嵌套子查询实现;
   select * from
   (select rownum r,t.* from
   (select * from 表名 order by 列名 desc) t
   where rownum<=20 )
   where r>10
